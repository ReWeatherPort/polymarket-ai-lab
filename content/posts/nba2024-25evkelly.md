# NBA 雷霆隊 2024-25 賽季投注實驗：EV 與 Kelly Criterion 實戰筆記

> 本文為個人實驗與學習紀錄，不構成任何投資或博彩建議。

## 1. 研究問題與背景

這篇筆記的核心問題是：如果我們把一支球隊（OKC Thunder）當成一個長期「資產」，在整個 82 場例行賽中，能否透過系統化的勝率估計 + EV 計算 + Kelly Criterion 資金管理，打出一個 **可重複、量化、可回測** 的投注策略？

選 OKC 的原因：
- 年輕、深度夠、2023-24 已打出 57 勝（57-25）。
- 2024-25 市場預測勝場普遍偏樂觀（如 BetMGM、FanDuel 等預測區間約在 58–62 勝附近）。
- 球隊節奏與攻防效率數據都偏穩定，適合做賽季級別的模型假設。

在這個實驗中，我假設：
- 目標是找出「一套可在未來賽季複製」的流程；
- 不追求預測得很神準，而是看 **整體組合的期望值（EV）和 ROI**；
- 透過 Kelly / Half Kelly 等不同倍率，觀察資金曲線及風險。

## 2. 資料與模型假設

### 2.1 勝場預測與隱含勝率

根據 2023-24 成績與 2024-25 市場預測資料，我將 OKC 2024-25 的「合理勝場」基準設定在 **62 勝** 左右，並建立一個近似的勝場分佈（以 62 為 mean、約 5–6 場為標準差的常態近似）。

再配合博彩市場常見讓分盤 / Moneyline 的賠率區間（例如 -150、-130、-110、-100 等），可以反推出對應的 **隱含勝率**：
- -150 → decimal 1.667 → implied p ≈ 60.0%
- -130 → decimal 1.769 → implied p ≈ 56.5%
- -110 → decimal 1.909 → implied p ≈ 52.4%
- -100 → decimal 2.000 → implied p ≈ 50.0%

模型的核心是：
1. 自己估計 OKC 在特定情境下的真實勝率 p_model；
2. 判斷市場給出的 p_market 是否低估（即有正 EV）；
3. 只在 p_model 明顯大於 p_market 時下注。

### 2.2 back-to-back 與旅程方向

文獻顯示，NBA back-to-back（B2B）與旅程方向會明顯影響比賽表現，例如有研究指出東向、西向旅行及時差造成的「social jetlag」會導致勝率下滑。這裡引用過去研究作為啟發，將 B2B 場次簡化成幾個 penalty 係數：

- 非 B2B：基準勝率 p_base
- B2B 同時有長途移動：勝率下調 Δp（例如 -3% 至 -5%）
- B2B 但移動距離較短：勝率較小幅下調

實際模型中，我會用：

```text
p_effective = p_base × 調整係數（travel + rest + 對手強度）
```

目標不是精準預測單場，而是 **在 82 場樣本內，避免系統性過度樂觀**。

## 3. Expected Value（EV）計算框架

對每一場準備下注的比賽，使用標準 EV 公式：

```text
EV = p × (賠率 - 1) - (1 - p)
```

其中：
- p：自己估計的勝率 p_model；
- 賠率：十進制賠率（例如 1.909 對應美式 -110）；
- (賠率 - 1)：贏時的淨收益倍數；
- 1：輸時損失 1 單位本金。

當 EV > 0 時，理論上有正期望；
EV 愈大，代表 edge 愈大，但也要小心樣本數與模型誤差。

在 OKC 2024-25 的模擬中：
- 在 82 場中，只挑出 EV > 0 的場次下注；
- 以基準勝場 62 場估計，整體組合的平均 EV 約在 0.55 左右；
- 對應到整季 ROI 估計約 55%–60%（視下注策略與 Kelly 倍率而定）。

這裡有幾個概念：
- **單場 EV 正，不代表實際一定賺，只是長期平均趨勢向上**；
- **季中波動會很大，可能連輸 5–10 場都很正常**；
- 重點是整個系統在大量樣本下是否有 edge。

## 4. Kelly Criterion 資金管理

### 4.1 基本公式

對於每一場有 edge 的下注，Kelly Criterion 建議的下注比例為：

```text
f* = (bp - q) / b
```

其中：
- b：賠率的淨利倍數（例如賠率 1.909 → b = 0.909）；
- p：勝率；
- q = 1 - p：輸的機率；
- f*：推薦投入 bankroll 的比例。

若 f* > 0，理論上有正期望下注；
若 f* ≤ 0，代表沒有 edge 或是負 edge，應該放棄該場。

### 4.2 Full Kelly vs Half Kelly

Full Kelly 在理論上最大化長期對數資產成長，但在實務上波動非常大，容易造成心理壓力與資金回撤過深。因此這個實驗同時比較：

- Full Kelly（100% Kelly）
- Half Kelly（50% Kelly）
- Quarter Kelly（25% Kelly）

舉例：
- 假設初始資金 10,000，某場 f* = 0.16（16%）
- Full Kelly：下注 1,600
- Half Kelly：下注 800
- Quarter Kelly：下注 400

模擬結果顯示：
- Full Kelly 的長期 ROI 最高，但最大回撤也最大；
- Half Kelly 在 **報酬 / 風險比** 上更平衡，資金曲線較平滑；
- Quarter Kelly 更保守，但容錯率高，適合對模型信心未滿時使用。

在 OKC 82 場實驗設定中：
- 以 10,000 起始本金、挑選 EV 為正的場次下注；
- Half Kelly 的最終資金比 Full Kelly 少一點，但最大回撤顯著降低；
- 若你是「會被 drawdown 影響心態」的操作者，Half Kelly 甚至 1/3 Kelly 會比 Full Kelly 更實際。

## 5. OKC 2024-25 賽季模擬結果摘要

> 下列為模擬與理論推演結果，並非真實歷史數據。

### 5.1 勝場分佈與 ROI

在以 62 勝為基準的模擬中：
- 中位數情境：約 58–60 勝；
- 悲觀情境：約落在 55 勝附近；
- 樂觀情境：可上探 65–68 勝甚至 70 勝以上（約 1-sigma 以外情境）。

配合 EV、Kelly 模型，若以 **EV > 0 的場次** 作為下注範圍：
- 總下注場次約為 60–70 場（視閾值設定）；
- 單場平均 EV 約 0.5–0.6；
- 整季理論 ROI 有機會落在 50%–80% 區間；
- 在極佳情境（模型預測接近實際 + 市場持續錯價）下，ROI 可突破 100%。

### 5.2 風險與 drawdown

實驗也特別觀察：
- 連輸 5–8 場的機率並不低；
- 在高 Kelly 倍率下，這種連輸會導致帳戶短期回撤 30–40%；
- 若改用 Half Kelly，最大回撤多數情況降到 20% 左右。

這反映幾個重點：
1. 「有 edge」≠ 「賺得很穩定」，波動一定存在；
2. 資金管理策略的選擇，實際上是決定你能不能活到長期；
3. 心理上能接受的 drawdown，是比理論上多賺幾% 更實際的優化目標。

## 6. 實驗設計與延伸方向

這次 OKC 賽季模擬，只是建立一個 **單隊、單季** 的框架，接下來可以往幾個方向延伸：

### 6.1 多隊組合 vs 單隊深耕

- 單隊深入：更熟悉輪換、傷病、教練傾向，模型能做更多情境調整（如 B2B、客場長途旅行）。
- 多隊組合：分散 idiosyncratic risk，讓 ROI 來源更平均，避免一隊爆炸拖累整季表現。

### 6.2 加入機器學習模型

目前實驗傾向「結構化假設 + 文獻參考 + 手動調參」。未來可考慮：

- 使用 XGBoost / LightGBM，輸入進攻/防守效率、pace、rest days、travel distance 等特徵；
- 以歷史 10+ 年 NBA 比賽做訓練，直接預測勝率；
- 再把模型輸出的 p 放進 EV / Kelly 管線。

這樣可以：
- 減少主觀調整的比重；
- 提高可複製性；
- 方便做「backtest 多賽季」的壓力測試。

### 6.3 與 Polymarket 類型市場的結合

雖然本文主題是 NBA，但實際上的架構：

```text
事件 → 估計真實機率 p → 市場賠率 / 價格 → 計算 EV → 資金管理
```

這套流程其實可以直接搬到 Polymarket 等預測市場上：

- 把「比賽勝負」換成「事件是否發生」；
- 把「賠率」換成「市場價格（如 0.42 USDC）」；
- Kelly 部分仍然可以沿用，只是 b 的定義稍微不同。

對於喜歡做 data-driven 預測的人來說，NBA 賽季是一個很好練習場：
- 大量樣本（每季 1,000+ 場比賽）
- 數據透明
- 模型驗證容易

這些技巧再移植到其他市場（政治預測、宏觀事件）時，可以重複利用。

## 7. 寫在最後：如何避免「故事大於數據」

投注或預測市場很容易掉進「敘事陷阱」：

- 某隊連勝 → 高估後續勝率；
- 媒體看好 → 以為市場一定錯價；
- 自己有持倉 → find confirming evidence only。

這篇 OKC 實驗筆記的用意，是給自己一個提醒：

1. **先寫下假設，再看數據有沒有打臉**；
2. **用 EV / Kelly 把直覺量化**，逼自己面對風險與波動；
3. **接受「就算有 edge，也會有很痛的 drawdown」這個現實**。
